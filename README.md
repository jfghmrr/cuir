# БАРС: автозаполнение домашнего задания из Excel

Утилита для учителей: читает Excel-расписание (один файл = одна группа) и автоматически
проставляет домашние задания в электронный журнал на `es.ciur.ru` на вкладку
**«Задание на следующий урок»** через Playwright. Уроки сопоставляются по **теме**
(а не по дате — даты могут расходиться).

## Установка (macOS / однократно)

Системный `python3` на macOS поставляется с устаревшим Tk 8.5, у которого
ttk-/tk-виджеты невидимы в Dark Mode. Используйте Python 3.12 от Homebrew —
у него Tk 9.0:

```bash
brew install python@3.12 python-tk@3.12
cd /Users/jfghmrr/ciur
/opt/homebrew/bin/python3.12 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m playwright install chromium
```

## Запуск

```bash
cd /Users/jfghmrr/ciur
source .venv/bin/activate
python app.py
```

1. Жмёте **Выбрать…** и указываете `.xls` (имя файла должно начинаться с класса —
   `9а-1.xls`, `7б.xls`).
2. Класс / Предмет / Группа подставятся автоматически. Можно поправить.
3. **Холостой прогон** включён по умолчанию — проходит весь сценарий, но ничего не сохраняет.
   Полезно для первого запуска: посмотреть, что нашлось, без побочных эффектов.
4. **Старт** → откроется окно браузера с БАРС. Войдите через Госуслуги (ЕСИА) и нажмите
   **«Я вошёл (продолжить)»**. Сессия (cookies / localStorage) хранятся в каталоге
   `chrome_profile/` — это полноценный персистентный профиль Chromium, поэтому при
   следующих запусках браузер откроется уже залогиненным. Когда сессия протухнет
   (через несколько дней/недель), снова попросит войти.
5. В логе по каждому уроку увидите статус: `filled` / `skipped_existing` / `skipped_no_homework`
   / `not_found` / `error`.

## Идемпотентность

Если на уроке в БАРС уже есть запись «Задание на следующий урок» — скрипт её не трогает
(`skipped_existing`). Запускать второй раз безопасно.

## Галочка «Урок проведен»

Если тема урока в Excel совпала с темой урока в БАРС, скрипт дополнительно проверяет
галочку **«Урок проведен»** на вкладке «Урок». Если она ещё не стоит — ставит и сохраняет.
Если уже стоит — не трогает. В сводке прогона счётчик `marked_conducted=N`.

## Что брать из Excel

- Предмет — `[0, 3]`
- Группа — извлекается regex'ом из `[4, 1]` (формат: `группы "1гр-англ" класса`)
- Учитель — `[50, 3]`
- Класс — из имени файла (`9а-1.xls` → `9 А`)
- Уроки — таблица от строки `«Число и месяц»`: дата `[0]`, тема `[2]`, ДЗ `[17]`

«Не задано» пишется в БАРС буквально. Пустая ячейка ДЗ — пропуск (нет смысла создавать
пустую запись).

## Где править селекторы

Все локаторы Playwright собраны в `bars.py` (грепайте `_LOCATORS_TODO`). Если что-то
не находится — запустите Playwright codegen рядом с открытым БАРС и подставьте точные
селекторы:

```bash
python -m playwright codegen https://es.ciur.ru/
```

## Файлы

- `app.py` — tkinter GUI и оркестрация
- `parser.py` — парсинг Excel
- `bars.py` — Playwright-сценарий
- `auth.py` — ручной вход через ЕСИА + сохранение сессии
- `config.py` — URL, пути, таймауты
- `auth_state.json` — сгенерированный stage сессии (в `.gitignore`)
